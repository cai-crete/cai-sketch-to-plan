## 1. 프로젝트 개요
- **프로젝트 명**: sketch-to-plan
- **주요 기능**: Gemini AI API를 활용하여 스케치를 도면/계획으로 변환하는 기능 (추정)
- **주요 기술 스택**: React, TypeScript, Vite

## 2. 작업 내역 및 현재 상태 (최근까지 진행된 내용)
- **환경 변수 구성**: `.env` 및 `.env.example` 파일 세팅 (`GEMINI_API_KEY`, `APP_URL` 등).
- **GitHub 연동 시도**: 로컬 저장소를 초기화(`git init`) 하고 원격 저장소(`origin`)에 코드를 Push하는 과정을 논의함 (용량 및 네트워크 문제로 인한 지연 이슈 확인).
- **Vercel 배포 세팅 준비**: `vercel.json` 등 배포 설정 파일 확인 및 자동화(CI/CD) 과정 정리.

## 3. 앞으로 진행해야 할 남은 작업 (TO-DO)
- [ ] **GitHub 푸시 완료**: 작업 중인 전체 코드를 `main` 브랜치에 성공적으로 Push하기.
- [ ] **Vercel 배포 연결**: GitHub에 코드가 정상적으로 올라간 후, Vercel 대시보드에 Repository를 가져와(Import) 라이브(Live) 웹사이트로 배포하기.
- [ ] **환경변수 세팅 (Vercel)**: Vercel 배포 시 `GEMINI_API_KEY` 등 필수 환경변수 등록하기.
- [ ] **기능 개발 및 테스트**: 배포된 환경 또는 로컬(`npm run dev`)에서 Gemini API 연동이 정상적으로 작동하는지 핵심 기능 테스트하기.
